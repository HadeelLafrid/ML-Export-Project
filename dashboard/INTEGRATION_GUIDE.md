# Integration Guide — Clustering & Classification

This guide explains how to integrate your clustering and classification results into the shared dashboard.

---

## How the Dashboard Works

```
                    ┌──────────────────────┐
                    │  world_trade_data_    │
                    │  features.csv (512MB) │
                    └──────────┬───────────┘
                               │
                    ┌──────────▼───────────┐
                    │  prepare_data.py     │
                    │  (pre-aggregates)    │
                    └──────────┬───────────┘
                               │
                    ┌──────────▼───────────┐
                    │  dashboard/app.py    │  ◄── Run this
                    │  (Dash web app)      │
                    └──────────────────────┘
```

`prepare_data.py` reads the large feature CSV + all result CSVs and creates small aggregate files that `app.py` loads instantly.

---

## Step 1 — Save Your Results

### Clustering
After running your clustering notebook, save your results as:

```
data/res/cluster_labels.csv
```

**Required columns:**

| Column        | Type   | Description                          | Example    |
|---------------|--------|--------------------------------------|------------|
| `importer`    | int    | Country code (matches feature data)  | 12         |
| `product`     | int    | Product HS code (matches feature)    | 101        |
| `cluster_id`  | int    | Cluster assignment (0, 1, 2, ...)    | 0          |
| `cluster_name`| string | Human-readable cluster label         | "High Demand Markets" |

> `cluster_name` is optional but recommended for better dashboard display.

### Classification
After running your classification notebook, save your results as:

```
data/res/classification_results.csv
```

**Required columns:**

| Column             | Type   | Description                                      | Example    |
|--------------------|--------|--------------------------------------------------|------------|
| `importer`         | int    | Country code                                     | 12         |
| `product`          | int    | Product HS code                                  | 101        |
| `opportunity_label`| string | One of: `High`, `Medium`, or `Low`               | High       |

> The forecast team already generates `opportunity_labels` from the gap analysis. Your classification results **replace** these with more accurate ML-based labels.

---

## Step 2 — Re-run Data Preparation

After saving your CSVs, run:

```bash
# From the project root
python dashboard/prepare_data.py
```

This will:
1. Read your new CSV files from `data/res/`
2. Merge them with the trade data
3. Create updated aggregate files in `data/res/dashboard/`
4. The dashboard will automatically detect and display your results

---

## Step 3 — Verify the Dashboard

Start the dashboard:

```bash
# Install dependencies (first time only)
pip install -r dashboard/requirements.txt

# Run the dashboard
python dashboard/app.py
```

Open [http://127.0.0.1:8050](http://127.0.0.1:8050) in your browser.

### What to check:

**If your results loaded correctly:**

- **Intelligence tab** → The "Pending" badges should change to "✅ Integrated"
- **Opportunities tab** → Should show your classification-based labels (High/Medium/Low)
- **Trade Explorer tab** → Filters and charts should work with your updated data

### Troubleshooting

| Problem | Solution |
|---------|----------|
| Dashboard shows old data | Re-run `python dashboard/prepare_data.py` |
| Missing columns error | Check CSV column names match exactly |
| Empty intelligence tab | Check your CSV files exist in `data/res/` |
| App crashes on startup | Run `pip install -r dashboard/requirements.txt` |

## Git Workflow

```bash
# 1. Create your branch from main
git checkout main
git pull
git checkout -b feat/clustering-classification

# 2. Do your work, save CSVs

# 3. Commit your results (CSVs + notebooks)
git add data/res/cluster_labels.csv
git add data/res/classification_results.csv
git add notebooks/your_clustering_notebook.ipynb
git add notebooks/your_classification_notebook.ipynb
git commit -m "Add clustering and classification results"

# 4. Push and create PR
git push -u origin feat/clustering-classification
```

> **Tip:** Don't commit `world_trade_data_features.csv` (too large). It's already in `.gitignore`.

## Quick Check

Before submitting, run this to validate your files:

```python
import pandas as pd

# Check clustering
cl = pd.read_csv('data/res/cluster_labels.csv')
assert 'importer' in cl.columns, "Missing 'importer' column"
assert 'product' in cl.columns, "Missing 'product' column"
assert 'cluster_id' in cl.columns, "Missing 'cluster_id' column"
print(f"Clustering: {len(cl)} rows, {cl['cluster_id'].nunique()} clusters")

# Check classification  
cf = pd.read_csv('data/res/classification_results.csv')
assert 'importer' in cf.columns
assert 'product' in cf.columns
assert 'opportunity_label' in cf.columns
valid = {'High', 'Medium', 'Low'}
assert cf['opportunity_label'].isin(valid).all(), "Labels must be High/Medium/Low"
print(f"Classification: {len(cf)} rows")
print("✓ All good!")
```
